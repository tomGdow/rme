Notes
start value (whatwg)
The start attribute, if present, must be a valid integer.
It is used to determine the starting value of the list.
Six types of Automatic numbering
Decimal, Roman, Latin, Greek, Georgian, Armenian,
markup
decimal
decimal-leading-zero
lower-roman
upper-roman
lower-roman
lower-latin (lower-alpha)
upper-latin (upper-alpha)

armenian
georgian
latin vs alpha
upper-alpha and upper-latin are identical, as are lower-latin and lower-alpha
start
Value is always an Arabic number
type
a is lower-case letter
A is upper-case letter
i is Roman numeral (lower)
I is Upper-Roman
1 is decimal (default)
reversed
<ol reversed >
reverses the order
value vs start
value may be used instead of start as an attribute of the li tag, and overrides start in ul
(MeasureUp, but idiosyncratic?)
Rules for numbering calc (start/value)
The rules may be found at whatwg: li Element
  1. Let i be 1.
  2. If owner is an ol element, let numbering be owner's starting value. Otherwise, let numbering be 1
  3. Loop: If i is greater than the number of list items that owner owns, then return
  4. If item is an li element that has a value attribute, then:
    1. Let parsed be the result of parsing the value of the attribute as an integer.
    2. If parsed is not an error, then set numbering to parsed.
