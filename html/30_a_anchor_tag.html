anchor
The <a> element, with its href attribute, creates a hyperlink to webpages,files, email addresses, locations in the same page, or anything else a URL can address
Syntax
  1. code

Examples
  1. <a href="https://www.mozilla.com">Mozilla</a>
  2. <a href="#Section_further_down"> Jump to the heading below </a>
  3. <a href="mailto:nowhere@mozilla.org">Send email to nowhere</a>
  4. <a href="tel:+49.157.0156">+49 157 0156</a>
  5. <a href="http://www.company.com" target="iframe_inline"></a>
    <iframe src="products.html" name="iframe_inline"></iframe>
Notes
download attr
Prompts to save linked URL, instead of navigating to it
can be used with/without a value
<a href="" download="my_painting.png">
href attr
The URL that hyperlink points to
--section of webpage
telephone number with tel:
email addresses with mailto:
rel attr
The relationship of the linked URL as space-separated link types
target attribute
Where to display the linked URL, as the name of a browsing context
This may be a tab, a window or an iframe
_self is the current browsing context (default)
_blank usually opens a new tab, but user may decide to make open a new window
_parent is the parent browsing context (If no parent, behaves as _self )
_top is the topmost browsing context. In no ancestors, behaves as _self
Note on target
add rel="norefferer noopener" to stop possible exploitation.
With target=_blank, the above will also stop JS on target page running in same process as anchor-containing page.
hreflang
The hreflang attribute specifies the language of the document in the link
The attribute is purely advisory
<a href="https://www.w3schools.com" hreflang="en">W3Schools</a>
Browsing context
A browsing context is an environment in which Document objects are presented to the user
See whatwg: browsing context
