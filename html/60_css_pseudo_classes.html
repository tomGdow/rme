Pseudo Class and Pseudo Element
xx
Pseudo class
A CSS pseudo-class is a keyword added to a selector that specifies a special state of the selected element(s).
Denoted :
Pseudo element
A CSS pseudo-element is a keyword added to a selector that lets you style a specific part of the selected element(s)
Denoted ::
syntax
Pseudo Element
selector::pseudo-element {
  property: value;
}
Pseudo Class
selector:pseudo-class {
  property: value;
}

notes
Pseudo Element syntax
You can use only one pseudo-element in a selector.

It must appear after the simple selectors in the statement.

As a rule, double colons (::) should be used instead of a single colon (:).

This distinguishes pseudo-classes from pseudo-elements.

However, since this distinction was not present in older versions of the W3C spec, most browsers support both syntaxes.

Pseudo Class syntax
Like regular classes, you can chain together as many pseudo-classes as you want in a selector.
:default
The :default CSS pseudo-class selects form elements that are the default in a group of related elements.
it may match button, input type="checkbox", input type="radio", and option elements:
A default option element is the first one with the selected attribute, or the first enabled option in DOM order
multiple selects can have more than one selected option, so all will match :default.
input type="checkbox" and input type="radio" match if they have the checked attribute.
button matches if it is a formâ€™s default submission button: the first button in DOM order that belongs to the form. This also applies to inputs types that submit forms, like image or submit.

:nth-child

matches elements based on their position in a group of siblings.

syntax

The nth-child pseudo-class is specified with a single argument that describes a pattern for matching element indices in a list of siblings.

Element indices are 1-based (but n is zero based)

odd

even

An+B Represents elements in a list whose indices match those found in a custom pattern of numbers, defined by An+B, where: A is an integer step size, Bis an integer offset, n is all positive integers, starting from 0. It can be read as the An+Bth element of a list.

examples

:nth-child(odd)or :nth-child(2n+1)

:nth-child(even) or tr:nth-child(2n)

:nth-child(7) Represents the seventh element.

:nth-child(5n) Represents elements 5, 10, 15, etc. The first one to be returned as a result of the formula is 0 [=5x0], resulting in a no-match, since the elements are indexed from 1, whereas n starts from 0.
