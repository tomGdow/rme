inline vs block vs Inline-block
xx
block
Expand horizontally to take up a whole line
Can apply vertical margins
Typically displayed with a newline both before and after the element
Always start on a new line
inline
Expand horizontally just enough to contain content
Examples are strong and em
Cannot apply vertical margins
inline-block
Like inline, but can apply vertical margins
inline-block is now considered legacy
Syntax
  1. code

Notes
HTML-5
The distinction of block-level vs. inline elements was used in HTML specifications up to 4.01.
In HTML5, this binary distinction is replaced with a more complex set of content categories (qv)
block
the "block-level" category doesn't directly correspond to any HTML5 content category
"block-level" and "inline" elements combined together correspond to the flow content in HTML5.
hr is block-level (but br is inline)
Inline
Theimg element is by default inline
Maybe use img { display: block; max-width: 100%; }
video is inline
Generally, inline elements may contain only data and other inline elements.
You can't put block elements inside inline elements.
inline elements cannot have top/bottom margins but can have right/left margin. (Contrast with padding) [net ninja]
inline elements can have padding on all four sides
display
Using display:block does not category and the content model of the element.
Even if the display of the span element is changed to "block", it still would not allow to nest a div element inside it.
inline roughly corresponds to the new HTML5 category of phrasing content
